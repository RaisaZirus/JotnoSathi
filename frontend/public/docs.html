Pitch Deck · Problem

A broken system serving 170 million people

Bangladesh's community health workers operate without clinical tools, real-time data, or outbreak intelligence.

🗣️
No Bangla clinical AI
Every existing health AI tool responds in English. Bangladesh's 13,000+ Shebikas work in Bangla — creating a complete language barrier for clinical decision support.
📡
No outbreak intelligence
A measles case in Mymensingh today doesn't update the district risk model tomorrow. Outbreak signals are invisible until they become crises.
💸
Infrastructure cost barrier
Existing digital health tools require expensive servers or connectivity beyond clinic WiFi. Rural deployment is practically impossible at scale.
🔄
No feedback loop
Shebika observations are siloed. There is no mechanism for frontline case data to feed back into a live epidemiological surveillance system.
Pitch Deck · Solution

JotnoSathi: three systems, one platform

The only tool that speaks Bangla, works on clinic WiFi, costs nothing to deploy, and feeds worker observations into a live epidemiological model.

🧠
Bangla triage AI
LLaMA 3.3 70B via Groq. 5,934 WHO/MSF protocol chunks via TF-IDF RAG. Structured Bangla clinical guidance in under 10 seconds — risk level, immediate action, referral decision, danger signs.
🗺️
Live district risk map
8 Bangladesh divisions scored across 5 disease models using 13 public datasets. Absolute epidemiological ceilings prevent score inflation. Updates every 30 seconds.
🔄
Self-learning feedback loop
Every triage auto-generates a field report. 5 outbreak reports → automatic model retrain. Rajshahi measles: 24.8 LOW at baseline → HIGH after 5 reports. The system gets smarter with every case.

Technical · AI Layer

RAG pipeline & AI design

Why TF-IDF over semantic embeddings?
Medical terminology is precise and consistent. "Pre-eclampsia" always means pre-eclampsia. TF-IDF keyword matching performs equivalently to semantic similarity for this domain — while using ~5MB RAM vs ~300MB for a HuggingFace embedding model. This is what makes the stack deployable on Render's free tier.
Why Bangla in the system prompt?
Post-translation of English LLM output via Google Translate garbled clinical terminology. Moving the language instruction into the system prompt forces LLaMA 3.3 70B to reason and respond in Bengali directly — producing natural clinical Bangla. Double reinforcement (system + user prompt both asserting Bengali) makes it reliable across all 5 disease domains.

Technical · Security & Privacy

Security model

🔐
Authentication
JWT tokens, 8-hour expiry, stored in localStorage. bcrypt password hashing pinned to 4.0.1. SQLite user DB not externally accessible.
🕵️
No PII stored
Only symptom text, division name, and anonymous worker_id stored. No patient names, no identification. Field reports contain no personal health data.
⚕️
Clinical safety
Mandatory ethical disclaimer every session. LLM role: "assisting, not diagnosing." Temperature 0.2 for consistency. WHO/MSF-grounded responses reduce hallucination risk.
🌐
CORS
FastAPI CORS middleware configured. allow_origins=["*"] for hackathon deployment — to be restricted to known domains in production.
