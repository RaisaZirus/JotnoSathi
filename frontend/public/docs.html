Overview

Three systems, one platform

JotnoSathi combines real-time triage, outbreak surveillance, and a self-learning risk model into a zero-cost deployable stack for frontline healthcare in Bangladesh.

🧠
Bangla triage AI
Workers describe symptoms in Bangla or English. LLaMA 3.3 70B via Groq returns structured clinical guidance — risk level, immediate action, referral decision, danger signs — entirely in Bengali script in under 10 seconds.
🗺️
Live district risk map
8 Bangladesh divisions scored across 5 disease models using 13 public datasets. Composite scores use absolute epidemiological ceilings — no max-normalisation inflation. Updates every 30 seconds.
🔄
Self-learning feedback loop
Every triage auto-generates a field report. When 5 outbreak reports accumulate for a disease in a division, the risk model retrains automatically — the system gets smarter with every case filed.
📋
Auto field reports
Triage silently generates a structured outbreak or registry report in the background — zero extra work for the Shebika. Outbreak diseases (dengue, measles) go to queue; chronic diseases go to registry.
🏥
Referral network
Live map of district and tertiary hospitals across all 8 divisions with contact info and specialties — ready for immediate referral routing from any triage result.
🔒
Ethics first
Mandatory disclaimer every session: "তুমি একজন সহায়তাকারী, রোগ নির্ণয় করছ না।" Cannot be skipped. Every response begins with the same principle — assists, never diagnoses.

RAG Pipeline

How triage works

From Bangla symptom input to structured clinical guidance in Bangla — every step of the pipeline.

01
Input
Shebika enters symptoms in Bangla or English. Bangla input is translated to English via Google Translator for retrieval.
→
02
Disease detection
55+ keyword rules (English + Bangla script) route to one of 5 domains. Priority: maternal → BP → diabetes → dengue → measles → general.
→
03
TF-IDF retrieval
BM25-style scoring over 5,934 WHO/MSF chunks. IDF weights pre-computed at startup. Top-3 chunks retrieved. ~5MB RAM total.
→
04
Prompt construction
WHO context + disease-specific Bangla prompt fragment + symptom text assembled. System prompt enforces full Bangla output. Temp 0.2.
→
05
LLM generation
LLaMA 3.3 70B via Groq API generates structured Bangla response: ঝুঁকির মাত্রা, তাৎক্ষণিক পদক্ষেপ, রেফারেল, কারণ, বিপদ চিহ্ন.
→
06
Auto report
Field report generated silently. Outbreak diseases → queue (threshold 5 → retrain). Chronic diseases → case registry.

Risk Model

Division risk scoring

Each division is scored independently across 5 disease domains using 13 public datasets. Composite scores use absolute epidemiological ceilings — not max-normalisation — so scores mean something real.

🦟
Dengue
Patient burden, test positivity, climate-driven mosquito risk. Weight: 28%. Ceiling: 65.
🔴
Measles
Active Bangladesh 2026 outbreak, vaccination gap, under-5 mortality. Weight: 22%. Ceiling: 60.
🤰
Maternal
ANC coverage gap, skilled birth attendance, infant mortality, anemia. Weight: 22%. Ceiling: 55.
🩸
Diabetes
Prevalence rate, urban weighting, NCD burden. Weight: 15%. Ceiling: 50.
💊
Hypertension
Hypertension rate, dietary salt proxy, comorbid diabetes risk. Weight: 13%. Ceiling: 50.
Retrain feedback loop
When 5 outbreak reports (dengue or measles) accumulate for a division, train_model.py --silent is triggered via subprocess. The updated risk_scores.json is reloaded into the FastAPI process. Demo: Rajshahi measles starts at 24.8 (LOW) — five field reports push it to HIGH with a visible score jump on the risk map.

Ethics & Safety

Designed to assist, not replace

Clinical safety is the primary design constraint — every architectural decision was made with it in mind.

⚠️
Mandatory disclaimer
Shown at every session start before app access — stored in sessionStorage, checked on every mount. Cannot be skipped. "You are assisting, not diagnosing."
🔐
No patient PII stored
Only symptom text, division name, and anonymous worker_id are stored. No names, no identification. Passwords bcrypt-hashed. JWT tokens expire after 8 hours.
📖
Grounded in WHO protocols
Every response is anchored to 5,934 WHO and MSF clinical guideline chunks. The RAG pipeline ensures guidance is protocol-based, not hallucinated.
🌡️
Low temperature inference
Temperature 0.2 for all clinical responses — maximises consistency and reduces creative hallucination for structured medical guidance output.
